# 半导体+新能源联合复盘框架

> 用途：把半导体和新能源放在同一个A股研究框架中比较，帮助判断主线、轮动、验证和风险。  
> 提醒：不构成投资建议；所有实时数据均需更新验证。

## 1. 两大方向的共通逻辑

半导体和新能源看似不同，但A股研究方法有很多共通点：

| 共通问题 | 半导体怎么体现 | 新能源怎么体现 |
|---|---|---|
| 产业链位置 | 设计、制造、设备、材料、封测、模组 | 资源、材料、电芯、设备、应用、运营 |
| 国产替代 | EDA、设备、材料、先进封装、AI芯片 | 电力设备、储能系统、锂电材料、核心设备 |
| 周期波动 | 存储周期、晶圆厂资本开支、封测稼动率 | 锂价、硅料、组件、电池材料价格 |
| 技术迭代 | HBM、Chiplet、CoWoS、硅光、CPO | 固态、钠电、HJT、BC、钙钛矿 |
| 订单验证 | 客户导入、设备验收、服务器订单 | 储能招标、风电核准、电网招标、车销量 |
| 小作文风险 | 概念映射、华为链、HBM映射 | 储能订单、锂价反转、固态电池概念 |

## 2. 轮动路径对比

### 半导体轮动

```text
AI算力/国产替代催化
-> 光模块、服务器、PCB、连接器等先动
-> HBM/存储、先进封装、设备材料扩散
-> EDA/IP、国产芯片、晶圆制造补涨
-> 业绩和订单验证后分化
```

关键验证：AI服务器订单、光模块客户、HBM配套、封测稼动率、设备订单、材料认证、收入占比，均需更新验证。

### 新能源轮动

```text
政策/价格/需求催化
-> 上游资源或终端需求先动
-> 中游材料、电池、储能系统修复
-> 设备端和低位小票扩散
-> 业绩验证后分化
```

关键验证：锂价、硅料价格、组件价格、储能中标价、车销量、光伏装机、风电招标、电网投资，均需更新验证。

## 3. 如何判断主线强弱

| 观察项 | 强主线特征 | 弱主题特征 |
|---|---|---|
| 龙头 | 龙头持续放量上行，分歧后能修复 | 龙头冲高回落，后排乱涨 |
| 数据 | 有公告、订单、价格、财报验证 | 只有小作文和概念映射 |
| 扩散 | 按产业链有序轮动 | 每天换题材，无清晰传导 |
| 财务 | 毛利率、收入、现金流有改善迹象 | 只有故事，没有业绩线索 |
| 风险 | 利空后仍有承接 | 澄清公告后逻辑证伪 |

## 4. 公司分析统一模板

| 字段 | 半导体示例写法 | 新能源示例写法 |
|---|---|---|
| 产业链位置 | 设备/材料/封测/设计/模组 | 资源/材料/电芯/设备/运营 |
| 主营产品 | 刻蚀设备、CMP材料、光模块、存储模组 | 锂盐、正极、储能系统、逆变器 |
| 利润来源 | 订单、客户导入、国产替代、稼动率 | 价格、加工费、排产、毛利率 |
| 受益情景 | AI算力、国产替代、存储周期 | 价格反转、需求恢复、政策催化 |
| 证伪条件 | 无客户、无订单、收入占比低 | 价格继续跌、订单不兑现、毛利率恶化 |
| 小作文验证 | 查公告、客户、订单、收入占比 | 查公告、招标、价格、排产、财报 |

## 6. 小作文转假设流程

1. 提取原始主张：例如“HBM方向要爆发”或“储能订单大增”。
2. 判断类型：订单、价格、技术、客户、政策、业绩。
3. 找产业链位置：它影响设计、封测、设备、材料，还是资源、电芯、系统、运营。
4. 列验证数据：公告、财报、价格、订单、招标、客户认证。
5. 设置证伪条件：没有订单、收入占比低、毛利率不改善、公告澄清。
6. 记录下一次复核时间：财报窗口、公告窗口、行业数据更新时间。
