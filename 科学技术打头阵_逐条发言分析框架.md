# 科学技术打头阵：逐条发言分析框架

> 用途：以股票小白视角学习“科学技术打头阵”帖子中的狼大及其他用户发言。本文只用于学习、复盘和知识整理，不构成投资建议、买卖建议或收益承诺。

## 参考资料整合范围

本框架吸收并统一了 4 类参考材料：

- `Floor101_狼大交易体系百科全帖_深度分析.docx`
  - 复盘表格法：涨跌停、涨跌家数、成交量、北向资金、主力资金、龙虎榜、梯队完整性、外围市场。
  - 股指贴水：IF/IH/IC/IM 多空变化、贴水扩大/收敛、黄白线预判。
  - 做 T 条件：市场环境、时间段、黄白线、量能、支撑压力、禁止操作情形。
  - 止盈止损：金字塔/倒金字塔、被动止盈、破位纪律、仓位分层。
  - 主线与细分：主线识别、资金容量、逻辑延展、产业链拆分。

- `borrowable_points_summary.md`
  - 八个分析模块：盘面分析、日内操作、技术分析、仓位风控、板块主线、预期差、交易心理、复盘方法。
  - 五条总原则：先定义退出条件、盘面分层、主线由持续资金确认、预期差重算赔率、复盘还原判断链。

- `agent_trading_viewpoints_v2.md`
  - 单条发言标注方式：来源、楼层、类型、原文摘录、可借鉴观点、置信度。
  - 高频主题：分仓止损、事件前收缩仓位、北向回补、盘口分化、底部做 T、技术线代表资金预期、主线切换、预期落空后的认错。

- `分析揉搓线及日内总结.txt`
  - 揉搓线定义：一根 K 线同时有上影和下影，实体偏小，代表多空拉扯。
  - 核心判断：红 K/黑 K、上影/下影谁更长、当前是上涨趋势还是下跌趋势、是否配合量能。
  - 日内总结模板：市场定调、板块/个股调仓建议、核心博弈逻辑、风险/机会提示、论坛时段动态总结。

## 分析总原则

1. 先过滤，再分析。
   不是每条发言都值得长篇分析。交易信息少的互动发言保留原文和一句备注即可。

2. 先定义，再判断。
   小白不能直接记结论，必须先解释术语：主线、黄白线、量能、低吸、预期差、揉搓线、被动止盈、贴水等。

3. 盘面分层。
   指数、风格、板块、细分、个股、仓位、心理分别分析，不能把一个层面的结论套到另一个层面。

4. 数据验证。
   每条 A/B 类发言尽量补充当日指数、成交额、板块/个股表现；涉及个股简称时必须查全称和所属方向。

5. 不输出重复废话。
   删除固定化的“小白学习笔记”。学习点必须来自该条发言本身，不能每条重复相同句子。

6. 不把学习材料当交易指令。
   分析输出只还原判断链、条件和风险，不直接给买卖建议。

## 发言分级

| 类别 | 判断标准 | 处理方式 |
|---|---|---|
| A 类 | 涉及指数、板块、个股、资金、仓位、技术、预期、风险、做 T 等多个交易要素 | 完整分析 |
| B 类 | 只有少量交易关键词，或主要用于回答问题/解释术语/补充背景 | 中等分析 |
| C 类 | 闲聊、情绪、调侃、无有效交易信息 | 保留原文，简析一句 |

## 每条 A/B 类发言的标准分析结构

### 1. 元信息

- 时间
- 楼层
- 用户
- UID
- 发言时段：盘前、开盘确认段、早盘、午间、午后、尾盘、收盘后
- 分类：A/B/C

### 2. 原文清理

保留原文主体，去掉冗余论坛标签，但不改写核心表达。

### 3. 通俗解释

将原话翻译成小白能理解的条件判断：

- 这句话在说什么？
- 它是判断、计划、复盘、提醒、纠错，还是回答问题？
- 它针对的是指数、板块、个股、仓位，还是心态？
- 它隐含的动作倾向是什么：观察、低吸、持有、减仓、止损、做 T、不操作？

### 4. 当日市场环境

优先使用东方财富或腾讯财经接口补充：

- 上证指数、深证成指、创业板指、科创 50、中证 1000 涨跌幅。
- 上证成交额。
- 指数整体偏强、分化或偏弱。
- 若后续扩展分时数据，再补黄白线、涨跌家数、涨跌停数量。

分析要点：

- 如果指数偏弱，个股机会要看是否属于真主线核心。
- 如果指数强但板块分化，要分清权重护盘还是题材赚钱。
- 如果成交额放大，要判断资金流向主线还是杂乱轮动。

### 5. 板块与细分方向

必须拆到细分，不停留在大词：

| 大类 | 可继续拆分 |
|---|---|
| 核心科技/国产算力 | 华为昇腾、服务器、PCB、光模块、液冷、半导体设备、封测、EDA |
| 光通信/光模块 | CPO、光芯片、光器件、光上游材料、光模块龙头 |
| 半导体 | 设备、材料、封测、晶圆制造、EDA、算力芯片 |
| AI | 算力硬件、AI 应用、传媒游戏、大模型软件 |
| 有色金属 | 铜、铝、小金属、贵金属、光上游材料映射 |
| 消费 | 白酒、食品、旅游、后周期消费 |
| 金融 | 银行、券商、保险、权重护盘 |
| 医药 | 创新药、药店、器械、生物制品 |

判断板块阶段：

- 启动：低位异动，但持续性未确认。
- 确认：板块核心开始持续走强，资金愿意反复回流。
- 主升：主线资金集中，核心和中军联动。
- 加速：大众开始看懂，风险同步上升。
- 分歧：冲高回落、内部强弱分化。
- 退潮：核心破位，补涨失败，资金撤离。
- 跌深反弹：位置低但不等于主线。

### 6. 个股简称补全

凡是出现个股代码、全称、简称、论坛黑话式简称，都尽量补全：

| 原文简称 | 输出格式 |
|---|---|
| 沪电 | 沪电股份（002463，PCB/服务器高速板） |
| 长电 | 长电科技（600584，先进封装/封测） |
| 中际 | 中际旭创（300308，光模块/CPO） |
| 新易盛 | 新易盛（300502，光模块/CPO） |
| 英子 | 英维克（002837，液冷/数据中心温控） |
| 寒武 | 寒武纪（688256，国产 AI 芯片/算力） |

规则：

- 先用内置映射处理高频简称。
- 内置映射没有时，调用东方财富搜索接口查名称、代码、市场。
- 若无法确认，不猜测，标记为“待人工确认”。
- 输出必须包含：原文简称、全称、代码、所属方向、来源。

### 7. 技术结构

技术不是预测工具，而是资金行为的外显。

检查项：

- 突破：是否放量，是否有核心标的和板块跟随。
- 回踩：是否缩量，是否守住关键位。
- 缺口：是否回补，是否有岛形风险。
- V 形：是否确认，不能主观幻想。
- 均线：是止损位、观察位、加仓位还是减仓位。
- 上影线：上方抛压或冲高兑现。
- 下影线：下方承接或资金试探。

揉搓线规则：

| 趋势 | 形态 | 含义 |
|---|---|---|
| 下跌趋势 | 黑 K，下影长接上影 | 可能只是下跌中继 |
| 下跌趋势 | 红 K，下影长接上影 | 支撑位震荡选方向 |
| 下跌趋势 | 红 K，上影更长 | 有资金抢反弹，但上方压力重 |
| 下跌趋势 | 黑 K，上影更长 | 可能短期止跌，但不是反转确认 |
| 上涨趋势 | 黑 K，下影更长 | 开始分歧，提高警惕 |
| 上涨趋势 | 红 K，下影更长 | 强势整理，可能继续新高 |
| 上涨趋势 | 红 K，上影更长 | 有承接但不适合追高 |
| 上涨趋势 | 黑 K，上影更长 | 承接弱，可能短期见顶或至少减仓观察 |

量能配合：

- 放量 + 揉搓线：多空博弈激烈，可能出方向。
- 缩量 + 揉搓线：参与意义不足，更偏观察。

### 8. 预期差

每条涉及消息、会议、政策、事件、传闻的发言都要问：

- 市场原本预期是什么？
- 资金是否已经提前埋伏？
- 消息落地是超预期、符合预期、不及预期，还是利好兑现？
- 当前价格位置是否已经反映预期？
- 新的赔率是否值得继续承担风险？

典型错误：

- 只看消息好坏，不看位置。
- 利好兑现后还追高。
- 利空落地但资金回补时仍按旧恐慌卖出。

### 9. 仓位和风控

必须把观点拆成条件和动作：

- 入场条件是什么？
- 加仓条件是什么？
- 减仓条件是什么？
- 止损条件是什么？
- 如果判断错了，何时认错？
- 当前是趋势仓、试错仓、体验仓、防守仓，还是观察仓？

原则：

- 先定义退出条件，再讨论收益空间。
- 仓位不是信心表达，而是风险预算。
- 知道可能被洗盘，不等于可以无视止损。
- 破位时执行预案，不是临场找理由。

### 10. 日内操作和做 T

做 T 分析顺序：

1. 今天是否适合做 T。
2. 指数是否处于箱体或可控环境。
3. 黄线是否强于白线。
4. 板块是否有持续资金。
5. 个股是否在支撑/压力附近。
6. 分时是否缩量转放量。
7. 时间段是否合适：9:45-10:00 或 14:00-14:30。
8. 是否触发不操作条件。

禁止做 T 的情形：

- 黄线快速下穿白线并放量。
- 黄白线交织且缩量。
- 白线快速上穿黄线但题材弱。
- 接近大指数压力位。
- 为了交易而交易。

### 11. 交易心理

不输出空泛心态建议，只分析情绪如何影响执行：

- 这条发言是在纠正贪婪、恐惧、怕踏空、不甘心，还是死扛？
- 是否把浮盈当成已实现利润？
- 是否想吃拥挤交易最后一口？
- 是否因为市场无聊而硬做？
- 是否用“可能洗盘”拖延止损？

### 12. 判断链还原

每条 A/B 类发言最后都还原成一条判断链：

```text
发言时间位置
→ 当时大盘环境
→ 板块/细分/个股层级
→ 技术和量能证据
→ 预期差或资金行为
→ 动作倾向
→ 风险和失效条件
```

## 文档排版要求

- 文档名使用帖子名：`科学技术打头阵_发言逐条分析_YYYY-MM.docx`。
- 日期分章，避免一个月内容堆在一起。
- 原文和分析分开。
- 狼大发言使用浅绿色标题栏和更大字号。
- 其他用户发言也使用同一分析框架，但样式保持简洁。
- C 类发言只做一句简析，避免文档臃肿。
- 不再输出重复的“小白学习笔记”。
